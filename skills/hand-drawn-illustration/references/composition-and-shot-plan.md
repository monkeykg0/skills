# 构图与镜头计划

## 决策顺序

1. 写出一句读者收获：“看完这张图，读者会明白……”。
2. 从锚定段落提取 2–6 个可画的具体名词和 1–3 个动作。
3. 为主体、原因、动作、阻碍和结果分配物件角色。
4. 选择能直接呈现关系的构图，不要先选模板再硬套内容。
5. 写出从 A 到 B 的阅读动线，并检查箭头方向和结果状态。
6. 删除不承担信息作用的装饰物。

## 构图模式

| 模式 | 适用内容 | 画面语法 |
| --- | --- | --- |
| 单焦点 | 核心观点、结论、封面 | 一个放大主体，辅助物缩小，四周留白 |
| 对照 | 前后变化、正确与错误、两种选择 | 左右或上下并置，共用基线，差异集中在关键物件 |
| 路径 | 步骤、演进、用户旅程 | 2–4 个节点，用连续路径串联，异常状态嵌入相关节点 |
| 因果链 | 原因、动作、结果 | 从原因物件到动作再到结果，箭头只表达真实因果 |
| 环境隐喻 | 困境、压力、机会、系统性问题 | 小主体置于大环境中，用尺度关系表达情绪或处境 |
| 物件隐喻 | 抽象概念、品牌观点 | 一个熟悉物件发生不寻常但可理解的变化 |
| 聚散 | 复杂到简单、筛选、归纳 | 多个输入汇入一个清晰结果，保持入口与出口可追踪 |
| 行动瞬间 | 人物故事、行为改变 | 人物处于动作中，动作对象和结果都可见 |

## 系列规划

- 按关键论点而不是按自然段机械拆图。
- 第一张建立主题，中央图片解释关键关系，最后一张呈现结果、行动或余韵。
- 连续两张不要使用完全相同的构图。
- 共享色板和线条，但为每张图选择不同主轮廓，避免系列看起来像复制粘贴。

## 自检问题

- 去掉所有标签后，画面关系是否仍然能看懂？
- 主体是否来自原文，而不是惯用的泛化符号？
- 每个箭头是否有明确起点、终点和语义？
- 是否能删掉一个物件而不损失信息？能则删除。
- 这张图与系列中其他图片是否在构图上有足够差异？
